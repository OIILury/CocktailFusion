Instructions pour un fichier unique
▼

Votre fichier CSV doit contenir les colonnes suivantes :

Informations principales du tweet
  • id : Identifiant unique du tweet
  • created_at : Date de création (format: YYYY-MM-DD HH:MM:SS)
  • published_time : Date de publication Timestamp Unix
  • user_id : Identifiant unique de l'utilisateur
  • user_name : Nom complet de l'utilisateur
  • user_screen_name : Nom d'utilisateur @username
  • text : Contenu du tweet
  • source : Source du tweet (Twitter ou Bluesky)
  • language : Langue du tweet
Localisation (optionnel)
  • coordinates_longitude : Longitude
  • coordinates_latitude : Latitude
Métriques et statistiques
  • possibly_sensitive : Contenu sensible (true/false)
  • retweet_count : Nombre de retweets
  • reply_count : Nombre de réponses
  • quote_count : Nombre de citations
Contenu enrichi
  • hashtags : Liste des hashtags séparés par des virgules
  • urls : Liste des URLs séparées par des virgules
Relations avec d'autres tweets
  • retweeted_tweet_id : ID du tweet retweeté si applicable
  • in_reply_to_tweet_id : ID du tweet auquel on répond si applicable
  • quoted_tweet_id : ID du tweet cité si applicable
Instructions pour fichiers multiples
▼

Créez un fichier CSV séparé pour chaque type de données :

Fichier principal
  • tweets.csv : Contient les tweets avec les colonnes :
    • id, created_at, published_time
    • user_id, user_name, user_screen_name
    • text, source, language
    • coordinates_longitude, coordinates_latitude
    • possibly_sensitive
    • retweet_count, reply_count, quote_count
Fichiers de contenu enrichi
  • hashtags.csv : Contient les hashtags avec les colonnes : tweet_id, hashtag
  • urls.csv : Contient les URLs avec les colonnes : tweet_id, url
Fichiers de relations
  • retweets.csv : Contient les retweets avec la colonne : retweeted_tweet_id
  • replies.csv : Contient les réponses avec la colonne : in_reply_to_tweet_id
  • quotes.csv : Contient les citations avec la colonne : quoted_tweet_id
